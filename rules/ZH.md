# 比特币彩票

## 介绍

**比特币彩票** 是一个完全依托比特币区块链的去中心化彩票系统。它不属于任何组织，仅依赖区块链存在。生成数字的方法是透明的，并且可以在这个 GitHub 仓库中查看。通过使用交易哈希和区块哈希生成数字，相同的输入将始终生成相同的数字。

用于生成彩票数字的代码可以在仓库中找到，您可以随时验证。该过程完全可审计，对所有参与者透明。

## 彩票规则

### 1. **时间表与开始**
- **开始日期**：第一次彩票将于 **2026年01月01日 00:00 UTC** 开始。
- **频率**：每次彩票抽奖将持续 **1,008 个区块**（大约 1 周）。
- **首个区块**：第一次抽奖将基于比特币区块 **930210** 的挖掘。

### 2. **参与与费用**
- **每个序列的费用**：每个序列 5,000 satoshi（0.00005 BTC）。
- **每个交易的最大序列数**：每次交易最多可输入 **10 个序列**。超出此限制的部分将不被考虑。
- **最低存款**：如果您的存款少于 5,000 satoshi，将被视为捐赠，您不会生成任何序列。
- **额外存款处理**：只有 5,000 satoshi 的倍数会被接受，最高接受 50,000 satoshi。例如，如果您存入 9,999 satoshi，将只生成 **1 个序列**，剩余的 4,999 satoshi 将视为捐赠。
- **交易费用**：请确保检查钱包收取的交易费用。根据您的钱包设置，费用可能会从您的存款中扣除，这可能影响您的参赛资格。请确保存款在扣除交易费用后至少为 5,000 satoshi。

### 3. **数字生成**
彩票数字是通过您交易的 **交易 ID (TXID)** 和彩票期间最后一个区块的 **哈希值** 生成的。TXID 和区块哈希中的每个字符都映射到一个数字，从而生成 **1 到 50** 之间的数字。

您可以在此查看数字生成算法的代码：  
["../service/number_generation.js"]

### 4. **奖金分配（收入的 90%）**
- **60%**：分配给匹配 **5 个数字** 的用户。
- **20%**：分配给匹配 **4 个数字** 的用户。
- **10%**：结转到下一个彩票。
- **10%**：保留给年末的 **超级奖**。

如果没有参与者匹配 5 个数字，奖金将结转到下次抽奖。同样，4 个数字的奖金也会遵循此规则。

### 5. **透明度与审计**
- 彩票系统的源代码公开在 **GitHub** 上，任何人都可以下载并自行运行代码。

  - 使用 **交易 ID** 验证您的数字序列。
  - 查看以前的彩票结果和奖金。

您可以在这里找到这些页面：  
https://lotto-btc.com

### 6. **仓库结构**
仓库包含以下文件夹：
- **/reports**：包含每次彩票抽奖的报告，包括获奖者和奖金分配。
- **/prizes**：包含所有奖金支付记录，包括交易详情。
